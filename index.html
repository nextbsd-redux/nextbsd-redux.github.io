Why NextBSD

A modern foundation, born from open source.

Twenty-five years ago Apple bet the company on bringing NeXTSTEP — Mach, BSD, the Objective‑C runtime — into the next millennium. NextBSD picks up the other side of that lineage: take the system layer Apple open-sourced, run it on a living FreeBSD kernel, and finish the work in the open.

Mach where it belongs.

An out-of-tree mach.ko ports the Mach IPC subsystem onto stock FreeBSD-15 — no kernel patches, no forked tree. mach_msg works end-to-end in userland today.

launchd as PID 1.

One supervisor for boot, login, services, sockets, and timers. Declarative plists replace the rc.d hand-wave. Boot ordering becomes a graph, not a filename sort.

XPC for everything else.

libxpc + libdispatch give every daemon a typed, queued, multiplexed message bus — the same primitives the modern Apple system stack is built on, ported and ELF-native.

FreeBSD, all the way down.

ELF binaries. ZFS. jails. bhyve. Ports. pkg. None of it gets traded away. NextBSD adds a system layer on top of the FreeBSD you already know — it doesn’t replace it.

“Take the system layer Apple open-sourced, run it on a living FreeBSD kernel, and finish the work in the open.” — the project, in one sentence

The stack

What we’re porting.

Every component below is real Apple-open-source code being lifted onto FreeBSD, tracked in the open. No vapor — status reflects work-in-tree.

M

mach.ko

Out-of-tree Mach kernel module. mach_msg, ports, port rights, bootstrap server. Loads on stock FreeBSD-15; 19/19 smoke green.

Working
L

launchd

Apple’s launchd-842.92.1 — the last open-source release — ported onto our Mach + libxpc stack. PID 1 capable.

In progress
X

libxpc

Typed inter-process messaging on top of libdispatch + Mach. Forked from ravynOS; bootstrap server + connection lifecycle in flight.

In progress
D

libdispatch

Grand Central Dispatch with a native Mach backend for FreeBSD — DISPATCH_SOURCE_TYPE_MACH_RECV on kqueue.

In progress
C

CoreFoundation

Hybrid CF: GNUstep libs-corebase for value types; swift-corelibs CF for CFMachPort, CFRunLoop, CFBundle. Ships with Apple ICU.

In progress
N

notifyd / ASL

Lightweight named-event bus and Apple System Logger — structured log records, BSD-text dual output, same C API.

Planned
cfg

configd

One daemon owns network state, hostname, DNS. Apple’s configd ported as netconfigd with a Mach-RPC API.

Planned
io

IOKit registry

A hwregd that reads /dev/devctl, kldloads on nomatch, and exposes an IOKit-shape IORegistry. ioreg -l on FreeBSD.

Planned
fs

HFS+ / DiskArbitration

Apple’s HFS+ as a FreeBSD hfs.ko; DiskArbitration over libgeom + devctl(4). Foundation for native DMG mounting.

Where we are

Status & recent milestones.

NextBSD is built in public across several repositories. The work below is verifiable in commit logs — not roadmap promises.

  • 2026 · Q2
    freebsd-launchd-mach — Phase B Tier 1 + C1–C3 complete.

    10 Mach syscalls wired into FreeBSD’s reserved lkmnosys slots, lazy Mach init, end-to-end userland mach_msg proof. Zero kernel patches.

  • 2026 · Q1
    libicu vendoring landed.

    Swift Foundation ICU built as /usr/lib/system/libicucore.so, matching macOS’s libicucore.dylib layout.

  • 2025 · Q4
    gershwin-on-freebsd Phase 2 landed.

    launchd as PID 1 via Option D, bedrock plists from freebsd-launchd, gershwin-specific dshelper / loginwindow plists. No getty — LoginWindow is the login.

  • 2025 · Q3
    Live ISO booting on real hardware.

    cd9660 + mkuzip + gunion overlay + init_chroot pivot. 396 MiB minimal, 3.8 GiB with KDE Plasma 6.
